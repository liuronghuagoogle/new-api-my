要同步上游（原项目）的更新并升级你的部署，需要遵循一个标准的流程。因为你现在是采用 Docker + Git 的方式部署，步骤如下：

第一步：同步代码 (Merge Upstream)
你需要把原作者的最新代码合并到你的代码库里。

抓取上游更新：

git fetch upstream
切换到主分支（确认你在 main 分支）：

git checkout main
合并更新：

git merge upstream/main
注意：如果出现冲突（Conflict），需要手动解决文件冲突。如果没有冲突，通常会自动弹出一个编辑器让你确认提交信息（像刚才那样输入 :wq 回车即可）。
推送到你自己的 GitHub：

git push origin main
第二步：发布新版本镜像
代码合并后，你需要打一个新的 Tag 来触发 GitHub Actions 自动构建新镜像。

查看上游最新版本号： 通常在项目的 VERSION 文件里，或者你看原作者发布的 Release 版本号（比如 v0.3.0）。
打标签并推送（假设新版本是 v0.3.0，请替换成实际版本号）：

git tag v0.3.0
git push origin v0.3.0
这会触发 GitHub Actions 开始构建镜像 liuronghua35/new-api:v0.3.0
第三步：更新服务器部署
等 GitHub Actions 构建完成后（通常几分钟）：

修改 docker-compose.yml： 将里面的镜像版本号改成最新的：

image: liuronghua35/new-api:v0.3.0
提交并推送配置修改：

git add docker-compose.yml
git commit -m "chore: upgrade to v0.3.0"
git push origin main
登录服务器更新： ssh 登录到你的服务器，然后运行：

git pull
docker compose pull
docker compose up -d
这样你的服务就平滑升级到最新版本了！
